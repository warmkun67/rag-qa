\# RAG 智能问答系统



基于大模型 + RAG 检索增强生成技术构建的本地知识库问答系统，支持多格式文档自动解析、语义检索、精准问答，可应用于内部知识库、智能客服、文档助手等场景。



\## 技术栈

\- Python 3.9+

\- LangChain（RAG 流程编排）

\- FAISS（向量检索库，语义相似度匹配）

\- DeepSeek 大模型 API（LLM 推理）

\- Streamlit（可视化交互界面）

\- python-dotenv（环境变量管理）



\## 核心功能

\- ✅ 多格式文档加载与解析（支持 PDF / TXT）

\- ✅ 智能文本分块（递归字符切分，兼顾上下文完整性）

\- ✅ 语义检索（FAISS 向量库，精准匹配问题相关片段）

\- ✅ 低幻觉、高可靠问答（严格基于文档内容，禁止编造）

\- ✅ 网页可视化交互（左侧文档管理 + 右侧问答界面）

\- ✅ 文档动态管理（上传/删除文件，无需重启服务）

\- ✅ 对话历史管理（清空历史、复制答案）

\- ✅ 全局缓存优化（文档只加载一次，启动/响应速度大幅提升）



\## 项目结构

```

rag\_project/

├── docs/                # 知识库文档目录（存放 PDF/TXT）

├── .env                 # 环境变量配置（API Key 等）

├── config.py            # 全局配置文件

├── doc\_loader.py        # 文档加载与分块模块

├── rag\_chain.py         # RAG 检索链核心逻辑（FAISS 向量检索）

├── web.py               # Streamlit 网页端界面

├── app.py               # 命令行交互版本

├── run.bat              # 命令行一键启动脚本

├── run\_web.bat          # 网页端一键启动脚本

├── requirements.txt     # 项目依赖清单

└── README.md            # 项目说明文档

```



\## 环境准备

\### 1. 安装依赖

```bash

pip install -r requirements.txt

```



\### 2. 配置环境变量

在项目根目录创建 `.env` 文件，填写你的 DeepSeek API Key：

```env

OPENAI\_API\_KEY=你的DeepSeek\_API\_Key

```

（也可直接在 `config.py` 中配置）



\## 运行方式

\### 1. 网页版（推荐，可视化交互）

双击 `run\_web.bat` 一键启动，或执行命令：

```bash

streamlit run web.py

```

启动后访问 `http://localhost:8501` 即可使用。



\### 2. 命令行版

双击 `run.bat` 一键启动，或执行命令：

```bash

python app.py

```



\## 使用说明

\### 网页端操作

1\. \*\*文档管理\*\*：左侧边栏可上传/删除 PDF/TXT 文件，实时更新知识库

2\. \*\*问答交互\*\*：右侧输入问题，点击「开始回答」获取基于文档的精准回复

3\. \*\*辅助功能\*\*：支持清空对话历史、复制答案、查看引用来源



\### 命令行操作

直接输入问题，回车获取回答，输入 `exit` 退出程序。



\## 性能优化说明

\- 采用 FAISS 向量检索替代暴力匹配，检索效率提升 10 倍+

\- 全局缓存机制，文档仅加载一次，避免重复计算

\- 文本分块大小优化（512 字符），平衡检索精度与响应速度

\- 限制检索 Top-K 为 3，减少 LLM 计算压力，秒级响应
